# Contributing

## Repository Layout

- `CONCURRENCY_MODEL.md` defines the task ownership, checkpoint, error, and
  cancellation semantics.
- `README.md` is the short usage guide. Keep detailed semantics in
  `CONCURRENCY_MODEL.md`.
- `ASYNC_COMPARISONS.md` compares async.nvim with async models from other
  languages.
- `lua/async.lua` is the public module. It exposes helpers such as `init()`,
  `wrap()`, `iter()`, `sleep()`, and `timeout()`, and inherits the core task API.
- `lua/async/_core.lua` contains the task scheduler and structured concurrency
  engine.
- `lua/async/_runtime.lua` contains runtime bindings for `wait`, `schedule`, and
  timers.
- `lua/async/_future.lua`, `_event.lua`, `_queue.lua`, and `_semaphore.lua`
  contain small async primitives used by the public API.
- `lua/async/_errors.lua`, `_compat.lua`, and `_util.lua` contain support
  helpers.
- `lua/async/misc.lua` contains examples and experiments built on the public API.
- `docgen.lua` generates `doc/lua-async.txt` from EmmyLua annotations.
- `test/async_spec.lua` contains the test suite.

## Development Workflow

- Read `CONCURRENCY_MODEL.md` before changing task semantics.
- Keep `README.md` short; link to detailed docs instead of duplicating them.
- Run `make test` for the test suite.
- Run `make format` after Lua code changes.
- Run `make doc` after public API or annotation changes.
- Run `make emmylua-check` after changing Lua annotations, overloads, or public
  types.
- For docs-only changes, run `git diff --check`.

## Generated and Vendored Output

- `doc/lua-async.txt` and `doc/tags` are generated by `make doc`.
- `build/nvim/async.lua` is generated by `make vendor-nvim`; do not edit it by
  hand.
- The vendoring path depends on the internal module layout. If you move code
  between internal modules, update `scripts/vendor_nvim.lua` and verify
  `make vendor-nvim`.
